# **Documentación Técnica – Sistema Home Banking**

## **1. Backend – .NET 8**

### **1.1 Tecnologías Utilizadas**

* **.NET 8**: Plataforma de desarrollo para construir aplicaciones web y APIs.
* **C# **: Lenguaje de programación utilizado en el backend.
* **ASP.NET Core**: Framework para crear aplicaciones web y APIs RESTful.
* **Dapper**: Micro-ORM que facilita el acceso a bases de datos.
* **SQL Server Express 2019**: Sistema de gestión de bases de datos ligero y gratuito, utilizado para almacenar la información.

### **1.2 Arquitectura y Modelos**

La arquitectura sigue el patrón **Repository**, donde la lógica de acceso a datos se encuentra en un repository separado. Los modelos de dominio incluyen:

* **Cliente**: Representa al cliente del banco, con propiedades como `Id` y `Nombre`.
* **Cuenta**: Representa las cuentas del cliente, incluyendo el `Saldo` y un indicador de si es la cuenta principal.
* **Tarjeta**: Representa las tarjetas asociadas a una cuenta, con un número y un indicador de si es la tarjeta principal.
* **Movimiento**: Representa las transacciones realizadas con una tarjeta, incluyendo la `Fecha`, `Monto` y `Descripción`.

### **1.3 Respuesta de la API**

Los modelos de respuesta están diseñados para enviar al frontend información resumida:

* **MovimientoResumen**: Contiene la `Fecha`, `Monto` y `Descripción` de un movimiento, sin detalles internos como `Id` o `TarjetaId`.
* **ResumenCliente**: Agrupa el saldo de la cuenta principal y una lista de `MovimientoResumen`, proporcionando una vista clara y concisa para el usuario final.

### **1.4 Controladores y Endpoints**

El controlador `ClientesController` expone un endpoint RESTful que permite obtener el resumen del cliente. Este endpoint es asincrónico y se encarga de:

* Recibir el `clienteId` desde la solicitud.
* Consultar el repository para obtener el `ResumenCliente`.
* Devolver la información al frontend en formato JSON.

### **1.5 Configuración y Middleware**

La configuración incluye:

* **CORS**: Permite que el frontend (React) pueda comunicarse con el backend sin restricciones de origen.
* **Middleware de redirección**: En caso de una solicitud no encontrada (404), se redirige al usuario a la página principal del sitio (`https://gyf.com.ar/`).

## **2. Frontend – React + TypeScript + Vite**

### **2.1 Tecnologías Utilizadas**

* **React 19.2.4**: Biblioteca de JavaScript para construir interfaces de usuario.
* **TypeScript**: Superset de JavaScript que añade tipado estático, mejorando la robustez del código.
* **Vite**: Herramienta de construcción y desarrollo rápida para aplicaciones web modernas.

### **2.2 Configuración de TypeScript**

La configuración de TypeScript (`tsconfig.json`) asegura:

* Uso de `ES2022` y módulos `ESNext` para compatibilidad con las últimas características de JavaScript.
* Configuración estricta de tipos para evitar errores comunes.
* Compatibilidad con módulos JSON y soporte para JSX en React.

### **2.3 Estructura del Frontend**

El frontend está compuesto por:

* **Tipos**: Definición de interfaces como `Movimiento` y `ResumenCliente`, que estructuran los datos que se reciben del backend.
* **API**: Un archivo que encapsula las llamadas `fetch` hacia el backend, manejando la obtención del resumen del cliente y la gestión de errores.
* **Componentes**: Un componente principal `ResumenCliente` que permite al usuario ingresar su `clienteId`, solicitar el resumen y mostrar el saldo y movimientos, manejando también los casos de error y la visualización de datos.

### **2.4 Estilos**

El frontend incluye estilos básicos para la presentación de la información, asegurando una interfaz clara y amigable para el usuario.
